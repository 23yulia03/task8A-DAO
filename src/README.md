# Task8A-DAO - Управление продуктами и тегами

## Описание проекта

Проект представляет собой простое приложение для управления списком продуктов и их тегов. Продукты и теги хранятся в оперативной памяти в виде коллекций внутри класса `ListProduct`. Данные существуют только во время работы приложения и не сохраняются после его завершения.

## Основные функции

- **Добавление продукта**: Пользователь может добавить новый продукт, указав его название, количество и тег.
- **Редактирование продукта**: Пользователь может изменить название, количество или тег существующего продукта.
- **Удаление продукта**: Пользователь может удалить продукт из списка.
- **Добавление тега**: Пользователь может добавить новый тег для категоризации продуктов.
- **Удаление тега**: Пользователь может удалить тег из списка.

## Структура проекта

- **HelloController**: Контроллер для управления интерфейсом пользователя. Обрабатывает действия пользователя, такие как добавление, редактирование и удаление продуктов.
- **ListProduct**: Класс, который управляет списками продуктов и тегов. Хранит данные в оперативной памяти в виде коллекций.
- **Product**: Класс, представляющий продукт. Содержит информацию о названии, количестве и теге продукта.
- **Tag**: Класс, представляющий тег. Содержит информацию о названии тега.

## Как использовать

1. **Добавление продукта**:
  - Введите название продукта в поле "Название".
  - Введите количество продукта в поле "Количество".
  - Выберите тег из выпадающего списка.
  - Нажмите кнопку "Добавить".

2. **Редактирование продукта**:
  - Выберите продукт из таблицы.
  - Внесите изменения в поля "Название", "Количество" или выберите другой тег.
  - Нажмите кнопку "Редактировать".

3. **Удаление продукта**:
  - Выберите продукт из таблицы.
  - Нажмите кнопку "Удалить".

4. **Добавление тега**:
  - Введите название тега в соответствующее поле.
  - Нажмите кнопку "Добавить тег".

5. **Удаление тега**:
  - Выберите тег из выпадающего списка.
  - Нажмите кнопку "Удалить тег".

## Соответствие требованиям

- **Оперативная база данных**: Данные хранятся в оперативной памяти в виде коллекций внутри класса `ListProduct`. Это означает, что данные существуют только во время работы приложения и не сохраняются после его завершения.
- **Хранение в классе коллекции**: Все данные хранятся в коллекциях `ObservableList<Product>` и `ObservableList<Tag>` внутри класса `ListProduct`.
- **Поиск по параметру**: Поиск продуктов и тегов осуществляется по их идентификаторам и другим параметрам, таким как название и количество.
